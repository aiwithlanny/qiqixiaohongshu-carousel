# qiqixiaohongshu-carousel GitHub 首頁設計

## 目標

讓第一次看到專案的人在 5 秒內理解：這不是普通排版工具，而是一套先研究爆款、再優化選題與標題、最後用 Image2.0 生成小紅書圖文的完整工作流。首頁要降低安裝和使用門檻，並自然引導訪客下載與點 Star。

## 主要受眾

- 不懂技術、想節省小紅書營運時間的自媒體創作者。
- 已經使用 Codex，想把內容生產流程自動化的人。
- 有知識、產品、課程或服務，但不擅長包裝成高點擊圖文的人。

## 核心訊息

主標題：一句話，生成符合爆款邏輯的精美小紅書圖文。

副標題：先調研近期高互動筆記，再優化切入角度、標題與封面，確認排版後使用 Image2.0 生成成品；把原本需要 1–2 小時的調研、拆頁與排版濃縮成一次對話。

時間效益只作為流程目標，不保證每位使用者固定節省相同時間。

## 首頁結構

1. 首屏：專案名稱、主標題、副標題、平台與 Image2.0 標籤、Star 引導。
2. 效果展示：一張代表性封面，加上 3 張同系列內頁，讓訪客先看到成果。
3. 為什麼不是普通圖文工具：對比「直接排版」與「爆款調研 → 角度 → 標題 → 封面 → 生圖」。
4. 核心能力：近期爆款調研、5 個選題角度、對標連結、標題優化、封面拆解、排版確認、Image2.0 成品。
5. 工作流程：主題或文案 → 調研 → 人工提供封面截圖 → 確認標題與排版 → 生圖 → 品質檢查。
6. 兩種工作模式：只有主題；已有文案。
7. 快速安裝：先放最無腦的 Codex 對話式安裝，再提供手動安裝方式。
8. 快速使用：提供 3 個可以直接複製的自然語言範例。
9. 適合場景：知識型內容、AI 工具、自媒體教學、個人品牌、產品或課程推廣。
10. 輸入格式：文案或主題、圖片頁數、喜歡的風格；未填寫風格時由調研結果決定。
11. 更新方式：說明本機 Skill 與 GitHub 不會自動同步；更新後需再次發佈。
12. 常見問題：是否一定生圖、是否能直接給文案、為什麼需要封面截圖、是否保證爆款。
13. Star 引導：如果這套流程幫你省下調研與排版時間，請點 Star 支持更新。
